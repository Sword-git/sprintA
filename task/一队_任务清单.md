<details>
<summary>📄 参考代码</summary>

```python
from enum import Enum
from pydantic import BaseModel, Field
from typing import Optional, Dict, Any, List
from datetime import datetime
import uuid

class BoundaryType(str, Enum):
    TOPIC_COMPLETE = "topic_complete"
    TOPIC_SWITCH   = "topic_switch"
    TIMEOUT        = "timeout"
    FORCE          = "force"

class MemoryNode(BaseModel):
    """话题级记忆单元 —— 闸门放行后交付给存储层的最小结构"""
    id:            str              = Field(default_factory=lambda: str(uuid.uuid4()))
    topic_summary: str              = Field(..., description="话题摘要，由 LLM 生成")
    messages:      List[Dict[str, str]] = Field(..., description="原始对话窗口内容")
    boundary_type: BoundaryType     = Field(..., description="触发写入的边界类型")
    confidence:    float            = Field(..., ge=0.0, le=1.0, description="边界判定置信度")
    user_id:       Optional[str]    = Field(None, description="对齐 Mem0.add() 的 user_id")
    agent_id:      Optional[str]    = Field(None, description="对齐 Mem0.add() 的 agent_id")
    run_id:        Optional[str]    = Field(None, description="对齐 Mem0.add() 的 run_id")
    metadata:      Optional[Dict[str, Any]] = Field(None, description="扩展元数据")
    created_at:    datetime         = Field(default_factory=datetime.utcnow)
```
</details>

<summary>📄 参考代码</summary>

```python
from abc import ABC, abstractmethod

class MemoryRetrievedEvent(BaseModel):
    """检索事件 —— 记录本次检索的上下文，供后续话题连续性判定"""
    query:           str              = Field(..., description="用户查询内容")
    retrieved_nodes: List[MemoryNode] = Field(default_factory=list)
    context_window:  List[Dict[str, str]] = Field(default_factory=list)
    timestamp:       datetime         = Field(default_factory=datetime.utcnow)

class MemoryIngestionGate(ABC):
    @abstractmethod
    def process(self, message: Dict[str, str], **kwargs) -> bool: ...
    @abstractmethod
    def flush(self) -> List[MemoryNode]: ...
    @abstractmethod
    def get_state(self) -> Dict[str, Any]: ...
    @abstractmethod
    def reset(self) -> None: ...
```

### `B3-002` 实现 InterceptedMemory 子类

> 通过子类化 `Memory` 覆盖 `add()`，消息送入 `GateController.process_message()`

- **前置任务**：B3-001
- **状态**：🔴 未开始

**检验标准：**
- [ ] `infer=False` 时直接放行，不走语义判定
- [ ] `process_message` 返回 True 时调用原始 `add()` 批量写入
- [ ] `metadata` 透传并注入 `topic_summary` 和 `boundary_type`

<details>
<summary>📄 参考实现</summary>

```python
class InterceptedMemory(Memory):
    def __init__(self, gate_controller: GateController, **kwargs):
        super().__init__(**kwargs)
        self._gate = gate_controller
        self._original_add = super().add

    def add(self, messages, *, user_id=None, agent_id=None, run_id=None,
            metadata=None, infer=True, memory_type=None, prompt=None):
        if not infer:
            return self._original_add(messages, user_id=user_id, agent_id=agent_id,
                                      run_id=run_id, metadata=metadata, infer=False,
                                      memory_type=memory_type, prompt=prompt)

        msg_list = self._normalize_messages(messages)
        for msg in msg_list:
            should_flush = self._gate.process_message(msg, user_id=user_id,
                                                      agent_id=agent_id, run_id=run_id)
            if should_flush:
                flushed = self._gate.force_flush()
                for node in flushed:
                    self._original_add(node.messages, user_id=node.user_id,
                                       agent_id=node.agent_id, run_id=node.run_id,
                                       metadata={"topic_summary": node.topic_summary,
                                                 "boundary_type": node.boundary_type})
```
</details>

## 验收标准

| 标准 | 验证方法 | 通过条件 |
|:---|:---|:---|
| 🔒 **不污染** | 5 条同话题消息，验证 `add()` 未被调用 | 拦截调用次数 = 0 |
| 🔒 **不丢失** | 话题结束后验证所有消息写入 | `flush` 后 `size() == 0` 且 `add` 被调用 1 次 |
| 🔒 **不阻塞** | TTL=5s，10s 后检查 | `check_timeout()` 返回 True，`force_flush` 被执行 |
| 🔒 **可卸载** | `uninstall()` 后调用 `add()` | 行为与原生 `Memory.add()` 完全一致 |
| 🔒 **契约合规** | `MemoryNode.model_validate()` 校验 flush 输出 | 无 `ValidationError` |
| 🔒 **性能基线** | 对比原生 `add()` 延迟 | 拦截器额外开销 < 200ms（不含 LLM 判定时间） |

## 风险与依赖矩阵

| 风险项 | 影响 | 缓解措施 |
|:---|:---|:---|
| LLM 判定延迟高 | 用户感知卡顿 | 异步 `evaluate()`，process_message 先返回 False，后台判定后主动 flush |
| LLM JSON 解析不稳定 | flush 漏触发 | 容错解析 + 兜底启发式规则（消息数 > 10 强制判定） |
| 猴子补丁污染全局 | 其他测试受影响 | 优先子类化方案，install/uninstall 成对调用 |
| 与二/三队接口不对齐 | 联调延期 | Sprint 0 输出 mock 接口，并行开发 |
| Mem0 版本升级 add() 变更 | 拦截器失效 | 锁定 `mem0ai>=2.0.2,<2.1.0` |
